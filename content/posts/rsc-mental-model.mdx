## 왜 헷갈렸나

`'use client'`를 처음 접했을 때 가장 먼저 든 생각은 _"그럼 서버 컴포넌트는 언제 쓰는 거지?"_ 였다.

공식 문서는 명확하게 설명하지만, 막상 실제 코드를 짜다 보면 경계가 흐릿해진다.

## 핵심 관점 전환

컴포넌트 단위로 생각하지 말고 **경계(boundary)** 를 설계한다는 관점으로 바꿔야 한다.

```tsx
// 이렇게 생각하지 말고
function MyPage() { ... } // "서버야 클라이언트야?"

// 이렇게 생각하자
// 어디서 경계가 나뉘는가?
```

## 정리

- 데이터 페칭 → 서버
- 인터랙션 → 클라이언트
- 경계는 최대한 leaf 쪽으로 밀기
